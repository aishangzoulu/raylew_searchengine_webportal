网络爬虫

网络爬虫（又被称为网页蜘蛛，网络机器人，在FOAF社区中间，更经常的称为网页追逐者），是一种按照一定的规则，自动的抓取万维网信息的程序或者脚本。

中文分词

中文分词指的是将一个汉字序列切分成一个一个单独的词，也就是将连续的字序列按照一定的规范重新组合成词序列的过程。

索引引擎

索引种类

正排索引:网页和关键词的对应关系

倒排索引:关键词和网页的对应关系

索引存储

正排索引:关系型数据库，Mysql，数据库单表500W+

倒排索引:非关系型数据库，Redis，二进制文本文件近100M

索引查询

在搜索引擎运行后，将倒排索引全部加载到内存中。通过关键词查询倒排索引，可以快速得到对应的网页集合

爱搜的Web架构

一个好的搜索应用肯定是基于Web的，而且考虑到搜索引擎应用的海量数据和高并发特性，从软件工程的角度来说它应该是设计良好的。

  • SOA: 爱搜采用了SOA架构，爱搜的服务端以组件的形式实现的，主要有爬虫组件、预处理组件、数据库服务组件和AP I组件。
  • 客户端-服务端分离: 爱搜采用了客户端-服务端分离的架构，爱搜的服务端以webservice的形式对外暴露API，客户端以标准的接口和服务端进行 通信。
  • REST: 爱搜采用了REST架构，爱搜将系统中的任意对象都抽象为资源，允许用户以标准的HTTP请求形式对这些资源进行 操作。
  • 轻量级:爱搜采用自主开发的爬虫，全文索引引擎Renew，一切都说明了轻量级
  • 开源: 爱搜始终保持开放、分享和包容的态度，代码将托管在开源平台github上，如果你也喜欢搜索引擎，欢迎一起开发

爱搜整体采用客户端-服务端分离的SOA和RESTFul架构，方便扩展。爱搜是最轻量级的搜索引擎，可高度定制。来爱搜，用更好的搜索，遇见更好的自己。 爱搜.

Page Weight源于对搜索引擎网页排序进行一定的研究后，提出的一种基于Page Rank、TF-IDF和搜索统计等算法的混合排序算法，计算公式如下

其中,s1+s2+s3=1。PR即Page Rank，是网页的权威性，通过网页评级策略计算出来；BR即Basic Relation是网页与查询内容的基础相关性，可通过信息论中的TF-IDF来表示；SR是搜索相关性，表示用户主观的对网页的评判，可通过用户在一次搜索中对某些网页的点击行为计算。PW即Page Weight，用来表示网页的最终排名指标。s1,s2和s3表示各个子指标对PW的贡献比例。

该算法主要的难题是为s1,s2和s3确定更精确的区间，以提高PW的准确度。首先将s1,s2和s3确定在一个大致的区间，然后随机产生不同的参数向量，并对在向量s条件下的大量的用户搜索结果质量SQ进行模型评估。当搜索结果质量越好时，参数向量s也会收敛到一个较小的区间，此时算法运行结束，并固定向量s在周期内不再变化。
